        # Client Only

        API

        The Client-Only-component has a dedicated API documentation.

        The @vue-cdk/client-only-package contains a single component: ClientOnly. This component can be used to wrap non SSR friendly components.

        History & Credits

        @vue-cdk/client-only is a friendly fork of egoist/vue-client-only. The component was only slightly changed. However, a lot of tests and documentation was added to make it easier to consume. Those additions will be contributed back to the original authors. The original license of egoist/vue-client-only can be found within the source of @vue-cdk/client-only.

        # Installation

        $ npm install @vue-cdk/client-only --save

        # Usage

        After installing @vue-cdk/client-only you have to import the module and pass it over to Vue.use(â€¦). This will make the client-only-component globally available as CClientOnly.

        import Vue from 'vue'
import ClientOnly from '@vue-cdk/client-only'

Vue.use(ClientOnly)

        If you prefer locally registered components then please refer to the example below which is not using the @vue-cdk/client-only-plugin.

        # Examples

        <template>
  <div>
    <ClientOnly>
      <!-- this component will only be rendered on client-side -->
      <HelloWorld />
    </ClientOnly>
  </div>
</template>

<script>
import { ClientOnly } from '@vue-cdk/client-only'

export default {
  components: { ClientOnly }
}
</script>
